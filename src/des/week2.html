Git Branching Learning Tool: [Learn Git Branching](learngitbranching.js.org/) Git branching is very useful for when you are developing and coding. It allows you to creates sub versions of your final product ready to be reviewed. Once your code has be reviewed by someone else *(they might find ways to making your code easier to read and more modular)* then it can be pushed into your main branch. You can either use `git switch` or `git merge`. Git switch is looking to replace git merge in the future. Git merging is for merging branches into one another. You should not merge your main branch into another branch. Only merge branches into main. It is recommended not to use rebase.

To branch in Visual Studio Code use the command `git branch dev`.
To create a branch using Github, click the drop down and simply type in your new name branch
