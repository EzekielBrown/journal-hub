A user story is a short, general description of a feature written from the user's perspective. Its purpose is to describe the value a software feature will provide to the customer. It's tempting to think of user stories merely as software system requirements.

A storyboard is a tool that helps you visualize your project in advance. Projects starting with storyboards are less likely to make production blunders caused by miscommunication.
